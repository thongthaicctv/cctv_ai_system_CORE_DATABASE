II. khi camera sử dụng các ký tự đặc biệt trong mật khẩu ta cần sử dụng mã hoá để cho phần mềm kết nối.
        | Ký tự | Encode |
        | ----- | ------ |
        | @     | %40    |
        | :     | %3A    |
        | /     | %2F    |
        | #     | %23    |
        | ?     | %3F    |
        | &     | %26    |
        | space | %20    |

        => một số ví dụ:    camera                      Phần mềm    
                            Abcd1234@   =====>>>>>    Abcd1234%40
                            Abcd#1234   =====>>>>>    Abcd%231234
                            Abcd&1234   =====>>>>>    Abcd%261234
                            Abcd?1234@  =====>>>>>    Abcd%3F1234%40
                            Abcd 1234@   =====>>>>>   Abcd%201234%40

IV. cài database
nếutạo database bằng root thành công, nhưng user atg_app đang bị lỗi kiểu xác thực:

(2059, "Authentication plugin 'auth_gssapi_client' not configured")

Lỗi này không phải sai mật khẩu bình thường. Nó nghĩa là user atg_app đang bị MariaDB gán nhầm cơ chế đăng nhập auth_gssapi_client, trong khi PyMySQL/app không dùng kiểu này.

Cách xử lý: xóa user atg_app cũ và tạo lại bằng password thường.

1. Mở MariaDB Client bằng root

Đăng nhập bằng:

User: root
Password: mật khẩu cài MariaDB

Sau đó chạy nguyên khối SQL này:

DROP USER IF EXISTS 'atg_app'@'localhost';
DROP USER IF EXISTS 'atg_app'@'127.0.0.1';
DROP USER IF EXISTS 'atg_app'@'192.168.%';
DROP USER IF EXISTS 'atg_app'@'%';

CREATE USER 'atg_app'@'localhost' IDENTIFIED BY 'atg_password';
CREATE USER 'atg_app'@'127.0.0.1' IDENTIFIED BY 'atg_password';
CREATE USER 'atg_app'@'192.168.%' IDENTIFIED BY 'atg_password';
CREATE USER 'atg_app'@'%' IDENTIFIED BY 'atg_password';

GRANT SELECT, INSERT, UPDATE, DELETE
ON atg_order_system.*
TO 'atg_app'@'localhost';

GRANT SELECT, INSERT, UPDATE, DELETE
ON atg_order_system.*
TO 'atg_app'@'127.0.0.1';

GRANT SELECT, INSERT, UPDATE, DELETE
ON atg_order_system.*
TO 'atg_app'@'192.168.%';

GRANT SELECT, INSERT, UPDATE, DELETE
ON atg_order_system.*
TO 'atg_app'@'%';

FLUSH PRIVILEGES;

2. Kiểm tra plugin của user

Chạy tiếp:

SELECT User, Host, plugin
FROM mysql.user
WHERE User = 'atg_app';

Kết quả đúng nên là dạng:

atg_app | localhost | mysql_native_password
atg_app | 127.0.0.1 | mysql_native_password
atg_app | 192.168.% | mysql_native_password
atg_app | %         | mysql_native_password

Nếu vẫn thấy:

auth_gssapi_client

là file tạo user hoặc code Cài database đang tạo sai plugin.

V. chuyển database mặc định về ổ khác:
	-> mở MySQL clien (11.4.10-MariaDB)
	-> chạy: SHOW VARIABLES LIKE 'datadir'; dể tìm đường dẫn hiện tại
	-> Mở PowerShell/CMD bằng quyền Administrator: net stop MariaDB	=> báo The MariaDB service was stopped successfully.
	-> Copy toàn bộ thư mục data sang ổ mới ví dụ tạo D:\ATG_MARIADB_DATA => robocopy "C:\Program Files\MariaDB 11.4\data" "D:\ATG_MARIADB_DATA" /MIR /COPYALL
	-> Sửa file cấu hình MariaDB: thường C:\Program Files\MariaDB 11.4\data\my.ini hoặc C:\Program Files\MariaDB 11.4\my.ini => datadir=C:/Program Files/MariaDB 11.4/data sửa thành datadir=D:/ATG_MARIADB_DATA
	-> kiểm tra quyền hệ thống xem thư mục đã được cấp full quyền chưa?
	-> start lại  PowerShell/CMD bằng quyền Administrator: net start MariaDB sẽ có thông báo The MariaDB service was started successfully.
	=> kiểm tra lại bằng MySQL 
		SHOW VARIABLES LIKE 'datadir';
		USE atg_order_system;
		SHOW TABLES;
	

Cách an toàn hơn: backup/restore sang datadir mới
Nếu chưa quen di chuyển thư mục data, cách an toàn hơn là:
Máy cũ / datadir cũ
Backup:
"C:\Program Files\MariaDB 11.4\bin\mysqldump.exe" -u root -p --databases atg_order_system > D:\ATG_BACKUP_DB\atg_order_system_backup.sql
Cài lại MariaDB và chọn datadir mới
Khi cài lại, chọn:
D:\ATG_MARIADB_DATA
Restore
"C:\Program Files\MariaDB 11.4\bin\mysql.exe" -u root -p < D:\ATG_BACKUP_DB\atg_order_system_backup.sql
Cách này sạch và ít rủi ro hơn copy datadir thủ công.

Tốt nhất là chọn ngay khi cài MariaDB. Nếu đã cài rồi thì nên dùng cách backup → cài lại chọn datadir mới → restore, an toàn hơn tự copy thư mục data.
